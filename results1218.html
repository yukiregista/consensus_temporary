Simulation Settings¶

Simulation experiment on the performance of some consensus methods applied to Bayesian posterior draws.

Full Details (not changed from last setting)¶

Generation of “true” tree¶

  • Used birth-death model to create 1 tree topology with 100 taxa.
  • Tree height is first set to either 0.05 or 0.5, then multiplied with lognormal r.v. with mean 0, standard deviation 0.5.
  • Then, the root is removed and the tree is made unrooted.

True evolutionary model (1 models $\times$ 1 gamma rate heterogeneity)¶

  1. HKY model with transition/transversion ratio 2, and base frequencies of 0.15, 0.35, 0.35,0.15, for A, C, G, and T.
    1. gamma site substitution rate heterogeneity with alpha=beta=0.5

Simulated Sequece Data (2 different lengths)¶

  • Length: one of [100, 1000]
  • number of taxa: 100 (for fast computation)

Model for estimation (4 models)¶

we used Mr.Bayes to estimate with bayesian modeling.

  • The models are
    • GTR $\pm$ gamma rate heterogeneity
    • HKY $\pm$ gamma
  • The prior given follows default setting
  • We generate 120000 draws using mcmc.
    • The first 20000 draws are for burn-in (discarded).
    • One tree is preserved in every 100 draws. This gives samples of size 100000/100 = 1000.

Construction of consensus trees¶

  • Initial estimator:
    1. MAP estimate (topology with the maximum number of occurence in the posterior draws; In case of a tie, randomly one topology is picked.)
    2. majority rule consensus
    3. Rstar consensus rooted at the first taxa "S1"
    4. astral
    5. ASTRAL-LENGTHWEIGHTED

Metrics¶

For the time being, I only computed unweighted metrics.

Let $T, \hat{T}$ be the "true" tree and consensus tree. Let $B(T)$ to denote the set of bipartitions in the tree $T$.

  • Usual RF distance (symmetric difference)
    • $d_{sd}(T, \hat{T}) = e(T, \hat{T}) + e(\hat{T}, T)$, where $e(T_1,T_2) = \sum_{b\in B(T_2)} \mathbb{1}\left[ b\not\in B(T_1) \right]$
    • So the first term is the false positives and the second term is the false negatives
    • False positives and false negatives can be weighted as in Berry and Gascuel (1996), to give something like:
      $d_{sd, \alpha}(T, \hat{T}) = \alpha e(T, \hat{T}) + (1-\alpha) e(\hat{T}, T)$, for some $\alpha\in[0,1]$
  • "Symmetric transfer dissimilarlity"
    • Just using transfer distance instead of {0,1} distance in the definition of RF distance.
    • $d_{std}(T, \hat{T}) = f(T, \hat{T}) + f(\hat{T}, T)$, where $f(T_1,T_2) = \sum_{b\in B(T_2)} \frac{\phi(b, T_1)}{|b|-1}$ where $\phi(\cdot,\cdot)$ is the transfer index and $|b|$ is the size of the light side of $b$.
    • Can allow for a weight: $d_{std, \alpha}(T, \hat{T}) = \alpha f(T, \hat{T}) + (1-\alpha) f(\hat{T}, T)$ for $\alpha\in[0,1]$
  • Quartet distance
  • Distance between phylogenetic distance matrix
    • Each tree is represented as the distance matrix (between taxa, unweighted), and distance between trees are measured by distance of vectorized matrices.

Results¶

I've only added PTS_greedy and PTS_risk_greedy for the pruning method. These are added with the purpose of improving std loss, so please look at std column (maybe also symmetric_qdist).

Consensus methods (Rows)¶

  • maj_{model}_{pruning method} : majority consensus
  • rstar_{model}_S1_{pruning method}: Rstar consensus with rooting with leaf 1
  • MAP_{model}_{pruning method} : MAP estimate. Note that for sequence length 100, most of MAP appears only once in the posterior draws: In other words, all 1000 posterior draws were different, and one tree is randomly picked.
  • ASTRAL_{model}_{pruning method} : ASTRAL, without weighting.
  • ASTRALW_{model}_{pruning method} : ASTRAL, with weighting by branch length.

where

  • model: Model used for estimation. One of [GTR, GTRgamma, HKY, HKYgamma]
  • pruning method: One of
    • No thresholding
    • PS50: Edges are retained if posterior probability of them are more than 50%.
    • PTS50: Edges are retained if posterior transfer support (PTS) of them are more than 0.5.
    • PTS_greedy: Edges are pruned in ascending order of PTS starting from the one with smallest PTS until the risk improvement stops.
    • PTS_risk_greedy: Edges are pruned in a greedy way with respect to posterior risk based on PTS. The difference to the PTS_greedy is that it prunes the edge with highest risk reduction at each step, instead of looking only at the posterior support. It stops pruning when pruning of no remaining edges induce risk reduction. It does not add edges back after pruning.

Metrics (Columns)¶

  • sd, fp, fn : symmetric difference, and its decomposition to #false positives and #false negatives.
  • std, fp_tbe, fn_tbe : Transfer distance based metric (it's not bootstrap so we have to come up with a new name...), and its decomposition described in the metric section.
  • tqdist: quartet distance computed by tqDist. Corresponds to #false negatives of quartet topologies.
  • quartet_fp : #false positives of quartet topologies.
  • symmetric_qdist: symmetric quartet distance, defined by tqdist + quartet_fp

Some insights (from added result)¶

  • PTS_greedy does not perform better than PS50 in general (in terms of std). It indicates that greedy algorithm based only on posterior transfer support does not reach a point close to the optimal.
  • PTS_risk_greedy does perform better than any other pruning methods in terms of std. It indicates that greedy algorithm based on the actual risk reduction performs fine. It also is typically the best thresholding method in terms of symmetric quartet distance as well. (Although I have not yet implemented pruning algorithm that aims to optimize symmetric quartet distance.)

Some insights (from previous results, not changed)¶

  • Majority consensus performs very well in every "symmetric" metric (where false negatives and false positives are weighted equally, namely, sd, std, symmetric_qdist) when they are sufficiently resolved.
  • When majority fails to be largely resolved, in terms of symmetric quartet distance, ASTRAL and ASTRALW performs better.
    • There is a very slight tendency that ASTRALW performs better than ASTRAL in case of low signals, but needs much more detailed consideration.
  • MAP with 100 sequence length performs very terrribly..
    • The reason is that all 1000 posterior draws often have different topologies, so MAP ends up just randomly picking one tree form posterior.
      • Maybe the performance improves with more draws, but it would require very large amount of computation.
        • Maybe test with same number of iteration without thinning? (that would produce 100 * 1000 = 100000 draws...)
