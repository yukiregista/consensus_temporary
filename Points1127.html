Consensus methods to summarize posterior distribution

Majority rule consensus (common) and its problems

  • It has a theoretical justification: It is a Bayes estimator w.r.t. symmetric difference loss.
  • However, in the case of low signals with large number of taxa, majority-rule consensus could be very unresolved. This is the problem that we want to ease. I assume there are two main reasons for this:
    1. The number of possible branches (bipartitions) grow in exponential order, while the number of bipartitions in the tree is of linear order. This causes, for example, the following phenomenon
      • majority rule consensus may not recover the true tree from a distribution of trees centered at the true tree.
        • Ex. ‚Äúspheric‚Äù-type distribution P in the BHV tree space, centered at the true tree but with large enough variance ‚Üí majority rule consensus constructed from a sample from this distribution does not give true tree.
        • Maybe other distributions based on other distances would also lead to the same conclusion (even with symmetric difference?)
    1. When there is some error in the data (e.g. gene sequence from one taxon is very bad (in what sense?)) and some taxa are misplaced in the tree, branch support can get dramatically low.
      • This is the sort of the problem transfer bootstrap expectation can tackle (although we are not in the bootstrap situation).
        • Q: To produce this kind of situation, what kind of simulation setting can be considered?
          • randomly generating sequence for one taxa?
        • In this situation, TBE-based metric and symmetric difference should behave in a very different manner.

Use of quartets

  • The two problems listed above can be eased by looking at quartet discrepancies, rather than branch discrepancies.
    • When we compute symmetric difference and majority rule consensus, we extract at most n‚àí2n-2n‚àí2Ôªø internal edges from O(2n)O(2^n)O(2n)Ôªø possible edges.
    • However, when we use quartets, we extract at most (n4)\binom{n}{4}(4n‚Äã)Ôªø quartet topologies out of 3‚àó(n4)3*\binom{n}{4}3‚àó(4n‚Äã)Ôªø possible topologies. And for each quartet, we are always picking at most 1 topology out of 3 possible topologies regardless of nnnÔªø.
    • Intuitively, this gives more chances for each quartet to be accepted.
      • In fact, Q* always refines (if is not the same) the majority rule consensus tree: If a branch gets more than 50% support, all quartet topologies induced by that branch also has more than 50% support.
      • However, the extent Q* refines majority rule consensus depends on the ‚Äútree-likeliness‚Äù of selected set of quartet topologies‚Ä¶
    • In fact, each problem above can be eased as follows:
      1. If the distribution is in such a way that quartet topology in the ‚Äútrue‚Äù tree ab||cd has more mass than other conflicting quartet topologies (ac||bd and ad||bc), Q* recovers the true tree.
      1. Even if one taxon aaaÔªø is misplaced in the tree, quartet topologies that don‚Äôt include aaaÔªø is preserved: So we can expect that it reconstructs tree structures unrelated to aaaÔªø properly.
    • Overall, measuring quartet supports allow us to look into more details of the posterior draws, possibly utilizing the additional information to improve majority rule consensus

    • As noted above, majority rule consensus is Bayes estimator w.r.t. symmetric difference loss.
    • Q* does not have such interpretation.
