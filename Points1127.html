Consensus methods to summarize posterior distribution

Majority rule consensus (common) and its problems

  • It has a theoretical justification: It is a Bayes estimator w.r.t. symmetric difference loss.
  • However, in the case of low signals with large number of taxa, majority-rule consensus could be very unresolved. This is the problem that we want to ease. I assume there are two main reasons for this:
    1. The number of possible branches (bipartitions) grow in exponential order, while the number of bipartitions in the tree is of linear order. This causes, for example, the following phenomenon
      • majority rule consensus may not recover the true tree from a distribution of trees centered at the true tree.
        • Ex. “spheric”-type distribution P in the BHV tree space, centered at the true tree but with large enough variance → majority rule consensus constructed from a sample from this distribution does not give true tree.
        • Maybe other distributions based on other distances would also lead to the same conclusion (even with symmetric difference?)
    1. When there is some error in the data (e.g. gene sequence from one taxon is very bad (in what sense?)) and some taxa are misplaced in the tree, branch support can get dramatically low.
      • This is the sort of the problem transfer bootstrap expectation can tackle (although we are not in the bootstrap situation).
        • Q: To produce this kind of situation, what kind of simulation setting can be considered?
          • randomly generating sequence for one taxa?
        • In this situation, TBE-based metric and symmetric difference should behave in a very different manner.

Use of quartets

  • The two problems listed above can be eased by looking at quartet discrepancies, rather than branch discrepancies.
    • When we compute symmetric difference and majority rule consensus, we extract at most n−2n-2n−2﻿ internal edges from O(2n)O(2^n)O(2n)﻿ possible edges.
    • However, when we use quartets, we extract at most (n4)\binom{n}{4}(4n​)﻿ quartet topologies out of 3∗(n4)3*\binom{n}{4}3∗(4n​)﻿ possible topologies. And for each quartet, we are always picking at most 1 topology out of 3 possible topologies regardless of nnn﻿.
    • Intuitively, this gives more chances for each quartet to be accepted.
      • In fact, Q* always refines (if is not the same) the majority rule consensus tree: If a branch gets more than 50% support, all quartet topologies induced by that branch also has more than 50% support.
      • However, the extent Q* refines majority rule consensus depends on the “tree-likeliness” of selected set of quartet topologies…
    • In fact, each problem above can be eased as follows:
      1. If the distribution is in such a way that quartet topology in the “true” tree ab||cd has more mass than other conflicting quartet topologies (ac||bd and ad||bc), Q* recovers the true tree.
      1. Even if one taxon aaa﻿ is misplaced in the tree, quartet topologies that don’t include aaa﻿ is preserved: So we can expect that it reconstructs tree structures unrelated to aaa﻿ properly.
    • Overall, measuring quartet supports allow us to look into more details of the posterior draws, possibly utilizing the additional information to improve majority rule consensus
  • Theoretical justification?
    • As noted above, majority rule consensus is Bayes estimator w.r.t. symmetric difference loss.
    • Q* does not have such interpretation.
      • However, it is expected that Q* might perform better in terms of symmetric quartet distance than majority rule consensus. (next section)
      • Q: Suppose a consensus tree optimizes symmetric quartet distance. What kind of biological interpretations we can give? Is it preferred than symmetric difference in certain applications?

Q*, Conservative version of Q* and symmetric quartet distance

  • The majority rule consensus is Bayes estimator w.r.t. symmetric difference, because the following holds:
    • If a branch with more than 50% support is included in the consensus trees, it improves the risk. Namely, if branch support is xxx﻿, adding this edge will make risk improvement of x−(1−x)=2x−1x - (1-x) = 2x-1x−(1−x)=2x−1﻿. (So if x>0.5x>0.5x>0.5﻿, improvement is positive). Thus, on the other hand, a branch with less than 50% support worsens the risk.
    • So picking only branches with more than 50% support is the best. This is possible because these branches make up a tree.
  • Symmetric quartet distance is defined as the sum of symmetric difference for each quartet topology.
    • So the same observation holds: quartet topology with more than 50% support improves the risk, others worsen the risk.
    • However, unlike the previous case, the set of quartet topologies with more than 50% support (say Q50Q_{50}Q50​﻿) is not tree-like.
    • So the optimization problem to solve for Bayes estimator w.r.t. quartet distance is very complex: Find a tree-like set of quartet topologies AAA﻿ that minimizes the risk w.r.t. quartet distance.
      • We could maybe think about ASTRAL-type heuristic search?
    • Even though optimal estimator is hard to find, we can still construct a tree that corresponds to maximum tree-like subset of Q50Q_{50}Q50​﻿ using Q* algorithm.
      • we call this for now “conservative Q*”
      • In usual Q*, the maximum subset of QmajQ_{maj}Qmaj​﻿ is considered, where QmajQ_{maj}Qmaj​﻿ is the set of quartet topologies that appear more than the other two conflicting ones.
      • By the construction of Q*, we always know that Q* always refines (or is the same as) conservative Q*.
      • The optimal consensus w.r.t. symmetric quartet distance loss probably should include quartets with lower support as well (to include more quartet topologies with high support). So this “conservative Q*” is more conservative than the optimal one.
